IDEIA

Um jogo cooperativo/competitivo onde os jogadores precisam refatorar um sistema cheio de problemas antes que ele entre em colapso. Cada rodada traz code smells, e o time usa padrões de projeto e refatorações para resolver.

40 Cartas de Problema (Code Smells)
Ex: Long Method, Data Class, God Class, Feature Envy…

40 Cartas de Solução

20 Design Patterns (Strategy, Observer, Factory Method…)

20 Técnicas de Refatoração (Extract Method, Move Method…)

Medidor de Dívida Técnica (0–20)

Medidor de Qualidade do Software (0–10)

COMO FUNCIONA

A cada turno, compra-se 2 problemas.

O jogador tem 3 ações:

Aplicar um padrão

Aplicar uma refatoração

Descartar um smell (custa qualidade)

Se um problema permanecer por 3 rodadas → +2 dívida técnica.

Se dívida técnica = 20 → derrota.
